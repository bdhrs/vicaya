- Cross-check challenged DN18, but local resolve-citation confirmed the citation; adding SN45.28 made the point more robust.
- I bypassed the Obsidian CLI and wrote directly to the vault filesystem, treating that as equivalent to an Obsidian save. That was wrong for the skill: the CLI create/open path should be attempted first, and filesystem write should be an explicit fallback only.
- I generated a PDF using the embedded WeasyPrint helper instead of checking or using the user's existing PDF printer workflow. The PDF instruction appears stale or too prescriptive for this setup.

## What I'd change in the skill
- Add guidance for terminating hung YouTube transcript fetches and falling back to cached/manual video review.
- Add a reminder to search movement-internal term mappings such as *vossagga*, *paṭinissagga*, and *sampajāna* when evaluating modern method acronyms.
- Add a hard Phase 7 rule: always attempt `obsidian vault=... create ... open` first; direct filesystem write is fallback only after CLI failure, and the final report must explicitly say fallback was used.
- Clarify "Obsidian database" operationally as the Obsidian CLI create/open workflow, not merely writing a markdown file into the vault folder.
- Replace or qualify the WeasyPrint PDF section with the user's actual PDF printer/helper. Do not create ad-hoc PDF scripts when an existing printer workflow exists.
- Add cleanup guidance: temporary scripts under `temp/` must be deleted unless intentionally added to the repo.
- Add a Phase 7 verification step that confirms the note was created through Obsidian CLI, or records that only filesystem fallback occurred.
- Add a rule for user-named seed videos: if a seed video is long, cached, or described as a "starting point," mine its transcript before synthesis and extract explicit claim headings/timestamps into the scratch before writing the note.
- Split the skill into a short mandatory execution checklist plus optional reference sections. The current monolithic skill makes it too easy to lose the critical path.
- Add a "scope lock" before Phase 5: list the user-named seed sources and confirm each has been processed or explicitly deferred. This would have prevented missing the long Being Untangled transcript.
- Add a "write path" checklist that must be checked off in order: Obsidian CLI write, Obsidian CLI verification, PDF only via configured printer/helper, reflection, scratch update.
- Make PDF generation a named helper command, not an inline script in the skill. Inline scripts encourage ad-hoc files and drift from the user's actual workflow.
- Add a transcript-mining helper that can extract timestamped claim headings and keyword clusters from cached YouTube transcripts, especially auto-captions, without dumping huge JSON into the terminal.
- Add a "claim ledger" output mode for polemical/doctrinal investigations: source claim, exact timestamp/source, evidence tier, canon counter-source, status. This request needed a ledger more than a prose note.
- Add tool failure behavior for Calibre locks and YouTube hangs as operational rules, not scattered reminders.
- Design for weaker-agent failure modes rather than assuming a strong model will infer the critical path. The issue is not more prose; it is better control points.
- Add a mandatory "User-Named Seeds" gate before synthesis:
  `| Seed | Type | Processed? | Evidence extracted | If not, why |`.
  Synthesis cannot start until every seed is processed or explicitly deferred. This would have forced the Being Untangled long-video transcript pass.
- For doctrinal/polemical audits, make the claim ledger the first-class artifact:
  `| Claim | Source | Timestamp/page | Status | Canon counter-evidence | Next check |`.
  Prose findings should be derived from the ledger, not written before it.
- Add phase exit criteria:
  YouTube phase complete only when transcripts are fetched/mined or failure is recorded.
  Calibre phase complete only when lock status is checked and primary books are searched.
  Canon phase complete only when each claim has canon evidence, commentary evidence, or a logged no-hit search.
- Put a short mandatory execution checklist at the top of the skill, separate from the long reference:
  preserve user seeds; create scratch; process seeds before broad search; build claim ledger; search canon per claim; save via Obsidian CLI; verify via Obsidian CLI; reflect.
- Add a transcript-mining helper command, e.g. `transcript-claims VIDEO_ID --terms ... --window ...`, which outputs timestamped claim clusters from cached transcripts without dumping raw JSON.
- Replace inline operational recipes with configured helper commands. PDF generation in particular should be a named existing printer/helper, not an ad-hoc inline WeasyPrint script.
- Make the Obsidian write contract explicit and testable:
  primary write via `obsidian create/append`; verify via `obsidian file`; filesystem write only after CLI failure; final report must say which happened.
- Add an "agent failure checklist" before final response: did I overweight easy sources, underweight user seeds, stop after enough evidence, confuse artifact creation with workflow completion, or follow a stale secondary instruction mechanically?
